An object-oriented approach to concurrent programming

The Problem
  • Concurrent Hardware is becoming increasingly popular
  • But language support is inconsistent
    • Threads in Java and C++
    • Global Interpreter Lock in Ruby and Python
    • Actors in Scala and Io
    • CSP in Go
  • Some languages with built-in concurrency
    • Implicit parallelism in SISAL and Haskell
    • Actors in Erlang
